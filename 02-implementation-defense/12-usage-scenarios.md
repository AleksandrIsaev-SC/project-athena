# Сценарии использования приложения

## Группа 1: Основные пользовательские сценарии

### Сценарий 1: Регистрация и онбординг нового пользователя
**Актор:** Новый пользователь  
**Цель:** Быстро начать использовать приложение и получить персонализированный опыт

**Основной поток:**
1. Пользователь скачивает приложение из App Store/Google Play
2. Выбирает способ регистрации (email, социальные сети, аккаунт компании)
3. Заполняет базовую информацию (имя, возраст, пол)
4. Проходит спортивный онбординг:
   - Выбирает основные виды спорта
   - Устанавливает фитнес-цели (похудение, выносливость, сила)
   - Указывает текущий уровень подготовки
5. Настраивает предпочтения уведомлений
6. Получает персональные рекомендации для старта

**Альтернативные потоки:**
- A1: Существующий клиент компании → импорт данных о покупках
- A2: Приглашение от друга → автоматическое добавление в друзья
- A3: Прерванная регистрация → сохранение прогресса

### Сценарий 2: Запись и анализ беговой тренировки
**Актор:** Бегун-любитель  
**Цель:** Отследить прогресс и получить рекомендации по улучшению

**Основной поток:**
1. Пользователь открывает приложение перед пробежкой
2. Выбирает тип тренировки "Бег" и цель (интервалы, темповый, длительный)
3. Запускает запись, система автоматически подключает доступные датчики
4. Во время тренировки видит в реальном времени:
   - Темп, дистанцию, время
   - Пульсовые зоны
   - Пройденный маршрут на карте
5. При достижении целей получает аудио-подсказки
6. Завершает тренировку, данные автоматически сохраняются
7. Просматривает детальный анализ:
   - Сравнение с предыдущими пробежками
   - Рекомендации по восстановлению
   - Выявление трендов прогресса

### Сценарий 3: Создание йога-группы для регулярных занятий
**Актор:** Инструктор йоги  
**Цель:** Создать сообщество для совместных практик

**Основной поток:**
1. Пользователь создает новую группу "Утренняя йога"
2. Настраивает параметры группы (открытая/закрытая, геолокация)
3. Создает расписание регулярных онлайн-сессий
4. Приглашает участников через соцсети или поиск по интересам
5. Проводит первую сессию с трансляцией видео
6. Участники синхронно отслеживают активность
7. После сессии группа обсуждает результаты в чате

## Группа 2: Социальные взаимодействия

### Сценарий 4: Совместная вело-тренировка с друзьями
**Актор:** Велосипедист  
**Цель:** Тренироваться вместе удаленно

**Основной поток:**
1. Пользователь планирует вело-тренировку на выходные
2. Создает событие "Воскресный велозаезд" в приложении
3. Приглашает друзей из списка контактов
4. Участники присоединяются к событию
5. В назначенное время все одновременно начинают тренировку
6. В реальном времени видят прогресс друг друга на карте
7. Обмениваются голосовыми сообщениями во время заезда
8. По окончании сравнивают результаты и делятся достижениями

### Сценарий 5: Участие в глобальном челлендже
**Актор:** Пользователь, мотивированный соревнованиями  
**Цель:** Достичь цели в соревновательном формате

**Основной поток:**
1. Пользователь просматривает доступные челленджи в разделе "Соревнования"
2. Выбирает "Апрельский марафон - 100км за месяц"
3. Изучает условия и награды
4. Присоединяется к челленджу, система создает персональный план
5. Ежедневно отслеживает прогресс в лидерборде
6. Получает уведомления о достижениях других участников
7. По окончании получает виртуальный бейдж и скидку на продукты компании

### Сценарий 6: Поиск тренировочных партнеров поблизости
**Актор:** Пользователь в новом городе  
**Цель:** Найти единомышленников для совместных тренировок

**Основной поток:**
1. Пользователь активирует функцию "Найти рядом"
2. Система показывает активных пользователей в радиусе 5 км
3. Фильтрует по видам спорта и уровню подготовки
4. Просматривает профили потенциальных партнеров
5. Отправляет приглашение на совместную тренировку
6. Получает подтверждение и договаривается о месте
7. После тренировки обмениваются отзывами

## Группа 3: Коммерческие сценарии

### Сценарий 7: Автоматическое обновление беговой обуви
**Актор:** Регулярно тренирующийся бегун  
**Цель:** Своевременно заменить изношенную экипировку

**Основной поток:**
1. Система анализирует пробег текущей пары обуви (800+ км)
2. Отправляет push-уведомление: "Вашей обуви требуется замена"
3. Пользователь тапает уведомление → переход в каталог
4. Система рекомендует модели на основе:
   - Истории предыдущих покупок
   - Стиля бега и пронации
   - Предпочтений по брендам
5. Пользователь выбирает модель, просматривает отзывы
6. Совершает покупку в 1 клик (данные карты сохранены)
7. Получает трекинг доставки в приложении

### Сценарий 8: Персонализированные промо-акции
**Актор:** Пользователь, готовый к новым вызовам  
**Цель:** Получить релевантные предложения

**Основной поток:
1. Система анализирует активность пользователя (3 месяца регулярного бега)
2. Определяет готовность к марафонской дистанции
3. Предлагает пакет "Подготовка к марафону":
   - План тренировок
   - Рекомендуемая экипировка со скидкой
   - Регистрация на местный марафон
4. Пользователь принимает предложение
5. Автоматически применяются скидки к связанным продуктам

## Группа 4: Продвинутые сценарии

### Сценарий 9: Использование умных устройств для тренировки
**Актор:** Профессиональный спортсмен  
**Цель:** Максимально точный анализ показателей

**Основной поток:**
1. Пользователь подключает дополнительные датчики:
   - Пульсоксиметр
   - Лактатный анализатор
   - Монитор мощности для велосипеда
2. Настраивает отображение дополнительных метрик
3. Запускает специализированную тренировку
4. В реальном времени отслеживает комплексные показатели
5. После тренировки анализирует корреляции между метриками
6. Экспортирует данные для тренера

### Сценарий 10: Участие в виртуальном соревновании с про-атлетом
**Актор:** Любитель, стремящийся к профессиональному уровню  
**Цель:** Сравнить свои показатели с профессионалом

**Основной поток:**
1. Пользователь выбирает "Соревнование с атлетом"
2. Выбирает профессионального спортсмена из партнеров бренда
3. Запускает запись своей тренировки одновременно с записью про-атлета
4. В реальном времени видит сравнение темпов, мощности, техники
5. Получает персональные советы от AI-тренера на основе различий
6. После тренировки изучает детальный сравнительный анализ

### Сценарий 11: Семейные фитнес-соревнования
**Актор:** Родитель, мотивирующий семью  
**Цель:** Сделать фитнес семейной активностью

**Основной поток:**
1. Пользователь создает семейную группу
2. Настраивает общие цели (пройденные шаги, сожженные калории)
3. Каждый член семьи синхронизирует свои устройства
4. Система автоматически агрегирует активность всех участников
5. Еженедельный семейный лидерборд с призами
6. Родительский контроль за активностью детей

## Группа 5: Административные сценарии

### Сценарий 12: Модерация пользовательского контента
**Актор:** Модератор сообщества  
**Цель:** Поддержание качественного контента

**Основной поток:**
1. Система автоматически флагует потенциально нежелательный контент
2. Модератор получает уведомление в админ-панели
3. Просматривает контент в контексте
4. Принимает решение (одобрить, отклонить, ограничить)
5. В случае нарушений применяет санкции к пользователю
6. Система обучается на решениях модератора

### Сценарий 13: Аналитика эффективности промо-кампаний
**Актор:** Маркетолог компании  
**Цель:** Измерить ROI маркетинговых активностей

**Основной поток:**
1. Маркетолог создает промо-кампанию в админ-панели
2. Настраивает целевую аудиторию и триггеры
3. Кампания запускается автоматически для符合条件的 пользователей
4. Система отслеживает:
   - Показы и клики
   - Конверсии в покупки
   - Вовлеченность после покупки
5. Маркетолог анализирует эффективность в реальном времени
6. Корректирует кампанию на основе данных

## Группа 6: Сценарии интеграции

### Сценарий 14: Интеграция с корпоративными wellness-программами
**Актор:** HR менеджер компании-партнера  
**Цель:** Внедрить корпоративный фитнес-челлендж

**Основной поток:**
1. HR настраивает корпоративный аккаунт через API
2. Импортирует список сотрудников
3. Создает корпоративные челленджи и цели
4. Сотрудники присоединяются через корпоративный портал
5. Система агрегирует анонимизированную аналитику для HR
6. По окончании программа автоматически награждает лучших сотрудников

### Сценарий 15: Экспорт данных для медицинского анализа
**Актор:** Пользователь с медицинскими показаниями  
**Цель:** Предоставить данные врачу

**Основной поток:**
1. Пользователь активирует "Медицинский режим"
2. Настраивает расширенный сбор данных с согласия
3. После тренировочного цикла формирует отчет
4. Экспортирует данные в стандартизированном формате (FHIR)
5. Передает отчет лечащему врачу через защищенный канал
6. Получает рекомендации по коррекции тренировочного плана

## Группа 7: Сценарии восстановления и профилактики

### Сценарий 16: Восстановление после травмы
**Актор:** Пользователь с ограничениями по здоровью  
**Цель:** Безопасное возвращение к тренировкам

**Основной поток:**
1. Пользователь указывает тип травмы и ограничения
2. Система создает адаптированный план восстановления
3. Предлагает альтернативные виды активности
4. Контролирует интенсивность и прогресс
5. Предупреждает о потенциально опасных упражнениях
6. Синхронизируется с рекомендациями физиотерапевта

### Сценарий 17: Предотвращение выгорания и перетренированности
**Актор:** Активный пользователь с высокой нагрузкой  
**Цель:** Оптимизировать нагрузку для долгосрочного прогресса

**Основной поток:**
1. Система анализирует patterns активности и восстановления
2. Обнаруживает признаки перетренированности (снижение HRV, нарушения сна)
3. Рекомендует снизить интенсивность или добавить дни отдыха
4. Предлагает альтернативные активности (йога, медитация)
5. Мониторит показатели восстановления
6. Постепенно возвращает к обычному плану при нормализации показателей
